# Convencion de tests backend

Por cada archivo TypeScript del backend (`*.ts`), crear un archivo de test en esta carpeta con el formato:

- `test-nombrearchivo.ts`

Ejemplos:

- Si existe `src/backend/modules/users/service/user.service.ts`, crear `src/backend/test/test-user.service.ts`.
- Si existe `src/backend/modules/schedules/repository/schedule.repository.ts`, crear `src/backend/test/test-schedule.repository.ts`.
